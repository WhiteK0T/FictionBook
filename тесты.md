## Стратегия тестирования

### 1. Unit-тесты

Изолированные тесты отдельных классов.

**Пример:**
```java
@Test
void paragraphBuilderShouldMergeTextChunks() {
    ParagraphBuilder builder = new ParagraphBuilder();
    builder.appendText("Hello ");
    builder.appendText("world");
    
    Paragraph result = (Paragraph) builder.build();
    
    assertThat(result.elements())
        .containsExactly(new Text("Hello world"));
}
```

**Покрывают:**
- NodeBuilder'ы
- Санитайзеры
- ResourceResolver
- ParagraphStyle

### 2. Интеграционные тесты

Тесты взаимодействия нескольких компонентов.

**Пример:**
```java
@Test
void shouldParseNestedSections() throws Exception {
    String xml = """
        <FictionBook xmlns="...">
            <description>...</description>
            <body>
                <section id="part1">
                    <section id="ch1"><p>Текст</p></section>
                </section>
            </body>
        </FictionBook>
        """;
    
    FictionBookDto book = readFromString(xml);
    
    assertThat(book.bodies().get(0).sections().get(0).subSections())
        .hasSize(1);
}
```

**Покрывают:**
- Fb2BlockParser + Fb2BodyParser
- Fb2Reader + EncodingDetector
- Fb2Writer + SanitizerPipeline

### 3. Round-trip тесты ✅

Чтение → Запись → Чтение → Сравнение. Реализованы: `Fb2RoundTripTest` (байт-в-байт
фикспоинт), `CorpusRoundTripTest` (корпус реальных FB2/FB3 через `@TestFactory`),
`Fb3WriterTest` (write→read для FB3).

**Пример:**
```java
@Test
void roundTripShouldPreserveData() throws Exception {
    FictionBookDto original = FictionBookIO.read(source);
    
    FictionBookIO.write(original, tempFile);
    FictionBookDto reread = FictionBookIO.read(tempFile);
    
    assertThat(reread).usingRecursiveComparison()
                      .ignoringFieldsMatchingRegexes(".*rawXml.*")
                      .isEqualTo(original);
}
```

**Цель:** Гарантировать консистентность парсера и писателя.

### 4. Property-based тесты ✅

Случайная генерация входных данных через jqwik. Реализован `Fb2RoundTripPropertyTest`
(фикспоинт на случайных DTO-деревьях).

**Пример:**
```java
@Property(tries = 500)
void parsedBookShouldNeverHaveNullTextNodes(@ForAll @Fb2Structure FictionBookDto book) {
    BookWalker.walk(book, node -> {
        if (node instanceof Text t) {
            assertThat(t.value()).isNotBlank();
        }
    });
}
```

**Цель:** Найти edge cases, которые не покрыты unit-тестами.

### 5. Fuzz-тесты ✅

Случайные байты, битые XML, глубокая вложенность. Реализованы: `Fb2ReaderFuzzTest`
(детерминированные краевые случаи на `@Test`) и `Fb2ReaderFuzzPropertyTest` (jqwik,
2×500 прогонов). **Важно:** `@Property` держим в отдельном чисто-jqwik классе — в одном
классе с Jupiter-конструкциями (`@Test`/`@Nested`) jqwik молча не исполняет проперти.

**Цель:** Гарантировать, что парсер никогда не падает с NPE/OOM/StackOverflow.

```java
void shouldApplySanitizersByDefault() throws Exception {
```

## Fuzz-тесты

Контракт «прощающего чтения»: на любом входе ридер обязан вернуть непустой DTO либо
бросить `FictionBookException` — никогда `NPE` / голый `XMLStreamException` /
`StackOverflowError`.

### Fb2ReaderFuzzTest (детерминированные случаи, JUnit Jupiter)

**Файл:** `src/test/java/.../internal/reader/fb2/Fb2ReaderFuzzTest.java`

```
├── emptyInput / nulBytes / plainText / foreignXml   # отвергаются как FictionBookException
├── truncatedMidTag                                  # обрезанный FB2 отвергается
├── invalidBase64InBinary                            # битый base64 проглатывается, книга читается
├── deepButBoundedNesting                            # 200 секций — читается в DTO
└── pathologicallyDeepNesting                        # 50 000 секций — без StackOverflowError
```

### Fb2ReaderFuzzPropertyTest (рандомизированный фьюз, jqwik)

**Файл:** `src/test/java/.../internal/reader/fb2/Fb2ReaderFuzzPropertyTest.java`
Два `@Property(tries = 500)`:
- `randomBytesNeverThrowUncontrolled` — случайные байты (0–512 байт);
- `corruptedValidFb2NeverThrowUncontrolled` — мутации эталонного FB2 (обрезка / замена
  байта / вырезание куска / вставка мусора).

> ⚠️ **Почему два класса.** `@Property` держим в **отдельном чисто-jqwik классе** без
> Jupiter-аннотаций. Если смешать `@Property` с `@Test`/`@Nested` в одном классе, движок
> JUnit Jupiter «забирает» класс, а jqwik перестаёт исполнять свои проперти — они молча
> докладываются как `skipped`. Именно этот баг и вскрылся: проперти не гонялись, пока
> жили в одном классе с детерминированными `@Test`.

```java
@Property(tries = 500)
void randomBytesNeverThrowUncontrolled(@ForAll("randomBytes") byte[] data) {
    assertForgiving(data);   // непустой DTO ИЛИ FictionBookException — и ничего иного
}
```

## Troubleshooting

### Тест падает с "No SLF4J providers were found"

Это предупреждение, не ошибка. Чтобы убрать:

```kotlin
// build.gradle.kts
dependencies {
    testImplementation("org.slf4j:slf4j-nop:2.0.13")
}
```

### Тест падает с "expected X but was Y" из-за кодировки

Проверьте, что XML-строки в тестах используют `StandardCharsets.UTF_8`:

```java
byte[] bytes = xml.getBytes(StandardCharsets.UTF_8);
```

### StAX генерирует `<p/>` вместо `<p></p>`

Это валидный XML. Обновите ожидания в тестах:

```java
assertThat(xml).contains("<p/>");  // StAX оптимизирует пустые теги
```

### Base64 содержит нули в конце

Убедитесь, что в `Fb2Writer.writeBinary()`:
```java
byte[] toEncode = (read == buffer.length) 
    ? buffer 
    : Arrays.copyOf(buffer, read);
```

### Тесты не запускаются после рефакторинга

```bash
./gradlew clean test --rerun-tasks
```
