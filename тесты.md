## Стратегия тестирования

### 1. Unit-тесты

Изолированные тесты отдельных классов.

**Пример:**
```java
@Test
void paragraphBuilderShouldMergeTextChunks() {
    ParagraphBuilder builder = new ParagraphBuilder();
    builder.appendText("Hello ");
    builder.appendText("world");
    
    Paragraph result = (Paragraph) builder.build();
    
    assertThat(result.elements())
        .containsExactly(new Text("Hello world"));
}
```

**Покрывают:**
- NodeBuilder'ы
- Санитайзеры
- ResourceResolver
- ParagraphStyle

### 2. Интеграционные тесты

Тесты взаимодействия нескольких компонентов.

**Пример:**
```java
@Test
void shouldParseNestedSections() throws Exception {
    String xml = """
        <FictionBook xmlns="...">
            <description>...</description>
            <body>
                <section id="part1">
                    <section id="ch1"><p>Текст</p></section>
                </section>
            </body>
        </FictionBook>
        """;
    
    FictionBookDto book = readFromString(xml);
    
    assertThat(book.bodies().get(0).sections().get(0).subSections())
        .hasSize(1);
}
```

**Покрывают:**
- Fb2BlockParser + Fb2BodyParser
- Fb2Reader + EncodingDetector
- Fb2Writer + SanitizerPipeline

### 3. Round-trip тесты (запланированы)

Чтение → Запись → Чтение → Сравнение.

**Пример:**
```java
@Test
void roundTripShouldPreserveData() throws Exception {
    FictionBookDto original = FictionBookIO.read(source);
    
    FictionBookIO.write(original, tempFile);
    FictionBookDto reread = FictionBookIO.read(tempFile);
    
    assertThat(reread).usingRecursiveComparison()
                      .ignoringFieldsMatchingRegexes(".*rawXml.*")
                      .isEqualTo(original);
}
```

**Цель:** Гарантировать консистентность парсера и писателя.

### 4. Property-based тесты (запланированы)

Случайная генерация входных данных через jqwik.

**Пример:**
```java
@Property(tries = 500)
void parsedBookShouldNeverHaveNullTextNodes(@ForAll @Fb2Structure FictionBookDto book) {
    BookWalker.walk(book, node -> {
        if (node instanceof Text t) {
            assertThat(t.value()).isNotBlank();
        }
    });
}
```

**Цель:** Найти edge cases, которые не покрыты unit-тестами.

### 5. Fuzz-тесты (запланированы)

Случайные байты, битые XML, огромные файлы.

**Цель:** Гарантировать, что парсер никогда не падает с NPE/OOM/StackOverflow.

void sanitizersShouldBeIdempotent() {
    FictionBookDto book = createBookWithSection(
        new Section("ch1", List.of(),
            List.of(new Paragraph(List.of(new Text("Text")))),
            List.of(), Map.of()));

## Лучшие практики

### 1. Arrange-Act-Assert

```java
@Test
void example() {
    // Arrange (подготовка)
    String xml = "<p>Текст</p>";
    
    // Act (действие)
    BlockElement block = parseFirstBlock(xml, "p");
    
    // Assert (проверка)
    assertThat(block).isInstanceOf(Paragraph.class);
}
```

### 2. Осмысленные имена тестов

```java
// ✅ Хорошо
void shouldSkipUnknownInlineTagsInParagraph()
void shouldAutoWrapOrphanTextInTableCell()
void shouldThrowWhenDescriptionIsMissing()

// ❌ Плохо
void test1()
void testParse()
void testError()
```

### 3. Изоляция тестов

```java
@TempDir
Path tempDir;  // Каждый тест получает свою папку

@BeforeEach
void setUp() {
    reader = new Fb2Reader();  // Свежий экземпляр
}
```

### 4. Параметризованные тесты

```java
@ParameterizedTest
@ValueSource(strings = {"<p>", "<subtitle>", "<v>"})
void shouldParseParagraphLikeElements(String tag) {
    // Один тест для нескольких случаев
}
```

### 5. AssertJ для читаемости

```java
// ✅ Хорошо
assertThat(result.elements())
    .hasSize(3)
    .first()
    .isInstanceOf(Text.class)
    .extracting(e -> ((Text) e).value())
    .isEqualTo("Hello");

// ❌ Плохо
assertEquals(3, result.elements().size());
assertTrue(result.elements().get(0) instanceof Text);
assertEquals("Hello", ((Text) result.elements().get(0)).value());
```

## Troubleshooting

### Тест падает с "No SLF4J providers were found"

Это предупреждение, не ошибка. Чтобы убрать:

```kotlin
// build.gradle.kts
dependencies {
    testImplementation("org.slf4j:slf4j-nop:2.0.13")
}
```

### Тест падает с "expected X but was Y" из-за кодировки

Проверьте, что XML-строки в тестах используют `StandardCharsets.UTF_8`:

```java
byte[] bytes = xml.getBytes(StandardCharsets.UTF_8);
```

### StAX генерирует `<p/>` вместо `<p></p>`

Это валидный XML. Обновите ожидания в тестах:

```java
assertThat(xml).contains("<p/>");  // StAX оптимизирует пустые теги
```

### Base64 содержит нули в конце

Убедитесь, что в `Fb2Writer.writeBinary()`:
```java
byte[] toEncode = (read == buffer.length) 
    ? buffer 
    : Arrays.copyOf(buffer, read);
```

### Тесты не запускаются после рефакторинга

```bash
./gradlew clean test --rerun-tasks
```
