# Корпус книг для round-trip тестов

Сюда кладутся **реальные** файлы книг, которые `CorpusRoundTripTest`
автоматически прогоняет по циклу чтение → запись → чтение → запись.

Раскладка:

- `fb2/*.fb2` — файлы FictionBook 2.x (один XML)
- `fb3/*.fb3` — файлы FictionBook 3.0 (OPC/ZIP-контейнер)

(на деле тест ищет `*.fb2`/`*.fb3` рекурсивно в этой папке, так что
подпапки можно организовывать как удобно).

## Что проверяет тест

Главный инвариант библиотеки «прощающее чтение / строгая запись»:
после первой строгой записи книга выходит на **фикспоинт**. Для каждого
файла выполняется

```
dto1   = read(файл)        // прощающее чтение «как есть»
bytes1 = write(dto1)       // первая строгая запись (с санитайзингом)
dto2   = read(bytes1)
bytes2 = write(dto2)
assert bytes1 == bytes2    // байт-в-байт фикспоинт
```

Расхождение `bytes1 != bytes2` означает баг в reader или writer
(потеря узла, перестановка атрибутов, схлопывание текста и т.п.).

Если папка пуста или отсутствует — тест не падает, а сообщает,
куда положить файлы.

> Лицензии: кладите только книги, которые разрешено распространять
> (общественное достояние / собственные образцы). Для крупных файлов
> рассмотрите Git LFS.
